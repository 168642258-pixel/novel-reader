# 零基础拿到 APK · 超详细图文教程

本教程假设你 **完全没装过** Java、Android SDK、Android Studio，也 **没用过** Git 和 GitHub。
跟着做就行，每一步都有具体位置和截图说明。

整个流程约 **15–20 分钟**（其中大部分是等待下载和构建）。

---

## 第一阶段：注册 GitHub 账号（3 分钟）

GitHub 是全球最大的代码托管网站，我们要把项目代码传上去，让它的服务器免费帮我们编译 APK。

1. 浏览器打开 **https://github.com/signup**
2. 填写：
   - **邮箱**（你的常用邮箱）
   - **密码**（自己设一个，至少 15 位，或 8 位含字母数字）
   - **用户名**（英文+数字，这是你以后仓库地址的一部分，比如 `xiaoming`）
3. 点 **Create account**
4. GitHub 会发一封验证邮件到你的邮箱，打开邮件点里面的链接完成验证
5. 最后一页问 "What do you want to do"，随便选一个或跳过即可

> 如果你已经有 GitHub 账号，直接登录，跳到第二阶段。

---

## 第二阶段：在 GitHub 上创建空仓库（1 分钟）

1. 登录后，点页面右上角你的头像旁边的 **＋** → 选 **New repository**

2. 按下面这样填：
   - **Repository name**：填 `novel-reader`（或任意英文名，别用中文）
   - **Description**：可填可不填，比如 "轻阅小说阅读器"
   - **Public** 还是 **Private**：
     - 选 **Public**（公开）：任何人能看到代码，但能免费用 2000 分钟/月的构建额度，推荐
     - 选 **Private**（私有）：只有你能看，免费额度 500 分钟/月，也够用
   - ⚠️ **下面三个复选框全都不要勾**：
     - ❌ Add a README file
     - ❌ Add .gitignore
     - ❌ Choose a license
     - （勾了的话首次推送会冲突，处理起来麻烦）

3. 拉到最下面点绿色的 **Create repository** 按钮

4. 创建完会跳到一个页面，地址栏里的 URL 就是你仓库地址，形如：
   ```
   https://github.com/xiaoming/novel-reader
   ```
   **把这个地址记下来**，第四阶段要用。

---

## 第三阶段：在本机安装 Git（3 分钟）

Git 是把本地代码推送到 GitHub 的工具。

1. 浏览器打开 **https://git-scm.com/download/win**
2. 页面会自动开始下载 **Git-x.x.x-64-bit.exe**，等它下完
   - 如果没自动下载，点 "Click here to download manually"
3. 双击运行安装程序，**一路点 Next** 即可，所有选项保持默认：
   - 安装路径默认 `C:\Program Files\Git`
   - 组件全勾
   - 默认编辑器用 Vim（不影响使用）
   - PATH 环境选第二个 "Git from the command line and also from 3rd-party software"（默认就是）
   - 后面一路 Next 直到 Install
4. 装完点 Finish

5. **验证安装**：在桌面或任意文件夹 **右键**，应该能看到 **"Git Bash Here"** 选项，说明装好了。

> 装过 Git 的跳过这一阶段。

---

## 第四阶段：推送代码到 GitHub（3 分钟 + 你操作）

这一步把 `小说app` 文件夹里的代码上传到 GitHub。

1. 打开文件资源管理器，进入：
   ```
   C:\Users\Administrator\Desktop\小说app
   ```

2. 在这个文件夹的 **空白处右键** → 选 **"Git Bash Here"**
   - 会弹出一个黑色命令行窗口，标题是 "MINGW64"

3. 在窗口里输入下面这行命令，然后按回车：
   ```bash
   bash push-to-github.sh
   ```

4. 脚本开始运行，你会看到：
   ```
   ========================================
     轻阅 · 推送到 GitHub 触发云端构建
   ========================================

   [1/5] 初始化 git 仓库...
   [2/5] 添加文件并提交...
   [master (root-commit) xxx] 轻阅小说阅读器初始版本
        25 files changed, ...
   ```

5. 接着会提示你粘贴仓库地址：
   ```
   [3/5] 需要你的 GitHub 仓库地址
      示例: https://github.com/你的用户名/你的仓库名.git
      或:  git@github.com:你的用户名/你的仓库名.git

   请粘贴仓库地址（直接回车跳过，稍后手动 push）:
   ```
   把第二阶段记下的地址粘贴进去（注意末尾要有 `.git`），例如：
   ```
   https://github.com/xiaoming/novel-reader.git
   ```
   按回车。

6. **首次推送时会弹出 GitHub 登录窗口**：
   - 弹窗标题可能是 "Sign in to GitHub" 或浏览器自动打开
   - **推荐用浏览器方式**：点 "Sign in with your browser"
   - 浏览器里登录你的 GitHub 账号 → 点 **Authorize git-ecosystem**（或类似按钮）授权
   - 看到 "Authentication complete" 表示成功，可以关掉窗口

7. 推送开始，看到类似输出表示成功：
   ```
   Enumerating objects: 30, done.
   ...
   To https://github.com/xiaoming/novel-reader.git
    * [new branch]      main -> main
   branch 'main' set up to track 'origin/main'.

   ========================================
     推送成功！
   ========================================
     打开你的 GitHub 仓库 → Actions 标签页
     等待 Build APK 工作流跑完（约 5-8 分钟）
     完成后点击对应运行 → 页面底部 Artifacts 下载 novel-reader-debug-apk.zip
   ```

### 常见报错处理

**报错 1：`fatal: remote origin already exists`**
不影响，脚本已经做了 `remove`，正常。

**报错 2：`Authentication failed`**
- 重新执行 `bash push-to-github.sh`
- 推送时如果还报错，用浏览器登录方式重试
- 或者改用 Personal Access Token：GitHub 网页 → 右上头像 → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token → 勾选 `repo` 全部权限 → 生成后复制 token，推送时用户名填你的 GitHub 用户名，密码粘贴这个 token

**报错 3：`Updates were rejected because the remote contains work...`**
说明你创建仓库时勾了 README。用强制推送覆盖：
```bash
git push -u origin main --force
```

**报错 4：中文路径导致脚本乱码**
改用 PowerShell 手动执行：
```powershell
cd C:\Users\Administrator\Desktop\小说app
git init -b main
git add .
git commit -m "init"
git remote add origin https://github.com/你的用户名/novel-reader.git
git push -u origin main
```

## 第五阶段：在 GitHub 上等构建完成（5–8 分钟）

1. 浏览器打开你的仓库：`https://github.com/你的用户名/novel-reader`

2. 点仓库顶部的 **Actions** 标签（和 Code、Issues 并排的那个）

3. 你会看到左侧有一个工作流叫 **Build APK**，右边有一个黄色圆圈转动的运行记录：
   ```
   ● 轻阅小说阅读器初始版本
     Build APK #1 · Commit main · 状态：in progress
   ```

4. 黄色圆圈 = 正在构建。等它变成 **绿色对勾 ✓**（成功）。
   - 如果变成 **红色叉 ✗**，点进去看日志，把报错截图发我，我帮你修
   - 通常首次构建约 5–8 分钟（要下载 Android SDK 和依赖）

5. 构建中你可以点进去看实时日志，了解它在干什么：
   - `Set up job` — 启动云端虚拟机
   - `Check out code` — 拉取你的代码
   - `Set up JDK 17` — 安装 Java
   - `Set up Android SDK` — 安装 Android 开发工具
   - `Build Debug APK` — 真正编译，这一步最久
   - `Upload APK` — 把成品打包

---

## 第六阶段：下载 APK（1 分钟）

1. 构建成功后（绿色 ✓），**点进这次运行**（点那条黄色/绿色的记录标题）

2. 拉到页面 **最下方**，会看到 **Artifacts** 区域：
   ```
   Artifacts
   ┌─────────────────────────────┐
   │ 📦 novel-reader-debug-apk   │
   │   5.2 MB                     │
   │   [点击下载]                  │
   └─────────────────────────────┘
   ```

3. 点 **novel-reader-debug-apk**，浏览器会下载一个 zip 文件：
   ```
   novel-reader-debug-apk.zip
   ```

4. 解压这个 zip（右键 → 解压到当前文件夹），里面有一个文件：
   ```
   app-debug.apk   ← 这就是你的安装包！
   ```

## 第七阶段：把 APK 装到手机（2 分钟）

### 方法 A：数据线传输（最稳）
1. 用 USB 数据线连接手机和电脑
2. 手机通知栏下拉 → 点 "USB 充电" → 改成 **"传输文件"**（MTP）
3. 电脑里打开手机存储，把 `app-debug.apk` 复制到 `Download` 文件夹
4. 手机上打开 **文件管理器** → Download 文件夹 → 找到 `app-debug.apk` → 点击安装
5. 如果弹出 "出于安全禁止安装未知来源应用"：
   - 点 **设置** → 开启 **"允许从此来源安装应用"** → 返回继续安装
6. 安装完成 → 打开 **轻阅** → 出现空书架 → 开始用！

### 方法 B：微信/QQ 传给自己
1. 电脑微信登录 → 文件传输助手 → 把 `app-debug.apk` 拖进去发送
2. 手机微信打开 → 文件传输助手 → 点 apk → 下载 → 点击安装

### 方法 C：网盘
1. 把 apk 上传到百度网盘 / 微云
2. 手机端下载后安装

## 以后改了代码怎么办？

每次你（或我）改了 App 代码，想让 GitHub 重新编译：

1. 在 `小说app` 文件夹 **右键 → Git Bash Here**
2. 执行：
   ```bash
   bash push-to-github.sh
   ```
   （脚本会自动检测变更并提交，回车跳过仓库地址输入，直接推送）
3. 5–8 分钟后去 Actions 页面下载新 APK
