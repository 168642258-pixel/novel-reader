## CI 构建失败 - run 27896186836（已修复）
第 8 步「构建 Debug APK」失败。用户下载了 build-output-log.zip。
日志定位：`ReaderScreen.kt:602 None of the following candidates is applicable` for SpanStyle。
根因：**SpanStyle 没有 lineHeight 参数**（lineHeight 是 TextStyle 的参数，不是 SpanStyle 的）。
之前在 buildContent 的 SpanStyle 里传了 lineHeight = ...，Kotlin 找不到匹配构造函数。
修复：从 SpanStyle 移除 lineHeight，行距由外层 Text 的 TextStyle 控制（已设）。
教训：Compose 中 SpanStyle 和 TextStyle 参数集不同，lineHeight 只在 TextStyle 中存在。

## APK 装好导入小说后闪退（已修复）
用户反馈：导入小说后 App 闪退，且每次打开都闪退。
根因：Book 实体的 chaptersJson 字段存了整本小说所有章节内容（几十 MB JSON）。
书架页 LibraryViewModel.books 用 Flow 订阅 dao.observeAll()，会把所有书（含巨型 JSON）一次性拉进内存 → OOM/ANR → 闪退。
因为是首页，每次开 App 都崩。

修复：
1. 新增 BookSummary 数据类（不含 chaptersJson 大字段）
2. BookDao.observeSummaries() 用 SQL 投影只 SELECT 元信息列
3. LibraryViewModel/LibraryScreen 全部改用 BookSummary
4. ReaderViewModel 把 chaptersFromJson 解析移到 Dispatchers.IO（原本在 Main，大 JSON 会卡主线程 ANR）
5. 空章节保护：chapters.isEmpty() 时返回 Error 状态，避免 IndexOutOfBoundsException

提交 60d7de3，新 run 27896981719 监控中。
教训：Room 存大字段（>1MB）时，列表查询必须用投影避免全字段加载。
大字符串 JSON 解析要在 IO/Default dispatcher，不能在 Main。
